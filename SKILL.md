# 公众号写作工作室（Keke WeChat Article Studio）

> © 2026 孙可可（keke）。本项目采用 CC BY-NC 4.0 协议授权。允许学习和交流使用，禁止商业用途，转载请注明原作者。

## 这个 Skill 解决什么问题

用户想做公众号，但经常卡在：

- 不知道写什么选题
- 有想法但写不成完整文章
- 标题、开头和结构不知道怎么优化
- 配图、排版和发布步骤太费时间
- 公众号后台草稿箱操作繁琐

这个 Skill 的目标不是替用户做长期运营，而是把单篇公众号文章的发布流程压缩成一条清晰工作流：

```text
账号定位 → 选题确认 → 大纲 → 正文 → 标题优化 → 配图建议 → 排版 → 发布草稿箱准备
```

## 触发场景

当用户提出以下请求时，使用本 Skill：

- “帮我写一篇公众号文章”
- “我不知道公众号写什么”
- “帮我把这个想法变成公众号文章”
- “帮我做公众号排版”
- “帮我生成公众号标题/开头/配图建议”
- “帮我把公众号文章发布到草稿箱”
- “我想用 AI 提高公众号发布效率”

如果用户只是问泛泛的商业模式、账号定位或内容战略，不要无限扩展成咨询服务；优先把问题收束到“今天要发布的一篇文章”。

## 第一次使用：检查账号配置

先检查 Skill 目录中的 `USER-PROFILE.md`。

如果不存在，或关键字段为空、仍是占位符，请提醒用户：

> 你的账号配置还没填写。这一步很重要：没有定位，我只能输出通用内容。请先复制 `USER-PROFILE-TEMPLATE.md` 为 `USER-PROFILE.md`，填写账号定位、目标读者、表达风格、内容边界和公众号信息。

不要在账号配置缺失时强行写完整文章。可以先帮用户填写或完善配置。

## 工作流

### Step 1：读取账号定位

从 `USER-PROFILE.md` 中提取：

- 账号定位：决定选题方向和内容深度
- 目标读者：决定用词难度、案例选择和价值表达
- 表达风格：决定语气、句式和结构
- 内容边界：决定不写什么、避免什么风险
- 配图偏好：决定图片风格和封面建议

如用户本轮给了明确要求，本轮要求优先级最高。

### Step 2：确认选题

如果用户只有模糊方向，先给 3-5 个可写选题，并说明每个选题适合的读者和发布价值。

选题输出格式：

```text
1. 选题标题
适合读者：...
为什么值得写：...
文章角度：...
```

让用户选择一个；如果用户不想选择，直接推荐最适合的一篇并继续。

### Step 3：生成文章结构

为选定选题生成公众号文章大纲：

- 标题方向
- 开头钩子
- 3-4 个主体小节
- 每节核心观点
- 结尾行动引导

公众号文章应避免空泛鸡汤，优先使用具体场景、真实困惑、可执行建议。

### Step 4：写正文初稿

根据大纲生成文章初稿：

- 开头 1-2 段，引入问题和核心观点
- 主体 3-4 节，每节 2-4 段
- 适当加入金句或总结句
- 结尾收束，并给读者一个明确行动

默认字数：1200-2500 字。用户要求长文时可扩展到 3000 字左右。

### Step 5：标题与开头优化

给出 3-5 个标题版本，区分用途：

- 稳妥清晰版
- 情绪共鸣版
- 痛点直接版
- 收藏转发版
- 个人表达版

同时检查开头是否能让目标读者继续读下去。如开头太泛，重写为更具体的场景式开头。

### Step 6：配图建议

根据文章内容给出配图方案：

- 封面图建议
- 正文配图位置
- 每张图的画面描述
- 如需 ImageGen，给出可直接使用的图像提示词

如果用户不需要 AI 生成图，也可以给图库搜索关键词或截图/照片使用建议。

### Step 7：公众号排版

将文章整理为适合公众号后台复制的结构：

- 标题
- 摘要
- 正文
- 小标题层级
- 引用块/重点句
- 图片占位符
- 结尾引导

如需要生成 HTML，使用 `scripts/wechat_template.py` 或项目内模板能力。

### Step 8：发布草稿箱准备

如用户已配置公众号 API，可继续准备发布草稿箱：

1. 检查 `assets/wechat_config.json`
2. 检查封面图是否存在
3. 上传正文图片到微信素材库，获得 `mmbiz.qpic.cn` URL
4. 生成文章 JSON
5. 调用 `scripts/wechat_publish.py` 创建草稿

如果 API 配置缺失，只输出可复制到公众号后台的排版稿，不阻塞用户发布。

## 重要边界

- 不承诺自动发布正式文章，只准备草稿箱或可复制稿。
- 不绕过微信公众号平台规则。
- 不生成违法、侵权、洗稿、搬运或高风险内容。
- 用户提供的账号密钥、AppSecret 等敏感信息只用于本地配置，不应写入公开仓库。
- 如果涉及医疗、法律、金融等高风险主题，需要提醒用户自行核验。

## 常用脚本

### 环境检查

```bash
python <skill_dir>/scripts/setup.py
```

### 安装依赖

```bash
pip install -r <skill_dir>/requirements.txt
```

### 图片清洗

```bash
python <skill_dir>/scripts/process_images.py <illustrations_dir>
```

### 上传用户图片

```bash
python <skill_dir>/scripts/upload_user_image.py <图片路径或目录路径>
```

### 发布到草稿箱

```bash
python <skill_dir>/scripts/wechat_publish.py <工作目录>/article.json <封面图路径> --config <skill_dir>/assets/wechat_config.json
```

## 新手推荐交付格式

如果用户只是想快速体验，交付以下内容即可：

1. 一个确定选题
2. 一篇文章初稿
3. 3 个标题
4. 一组配图/封面建议
5. 一份可复制进公众号后台的排版稿
6. 一个发布前检查清单

这能验证产品最核心的价值：从“不知道写什么”到“今天能发一篇”。
